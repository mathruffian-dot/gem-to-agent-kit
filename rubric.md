# rubric.md — 決策大腦（分類與升級評估標準）

agent 依此判斷「每個 Gem 該怎麼升級」。

## 一、為什麼要升級：Gem 的 6 大結構性限制
1. 無狀態（每次重傳材料）
2. 輸出唯讀（吐不出可用檔案）
3. 不能執行程式
4. 生圖綁網頁、無法進工作流
5. 知識檔靜態、有容量上限
6. 單一工具、不能編排多步驟

## 二、升級到哪：Agent 的 4 大殺手級功能
① 流水線化　② 真實檔案進出　③ 生圖進工作流　④ Python 當工具（MCP）

## 三、核心設計原則（每次升級都套）
**固定「流程＋規格」（凍在 Skill / AGENTS.md / scripts），浮動「材料」（每次丟 `input/`，形態不限、可批次）。**
輸入不是一句「主題」，而是一包「材料」。

## 四、六大分類 + 升級價值
| 類 | 內容 | 升級價值 | 升級重點 |
|---|---|---|---|
| A 數學備課/評量 | 教案、審題、複習、出題 | 🔥🔥🔥 | OCR + 出題腳本 + `.docx/.xlsx` + 批次 |
| C 視覺解謎生圖 | 各類密碼/謎題卡 | 🔥🔥🔥 | **邏輯從模型搬到 Python（精算）** + 生圖 + 批次 |
| E 影音工具 | 字幕、影片摘要 | 🔥🔥🔥 | 真的能跑 whisper/yt-dlp/ffmpeg |
| B 觀課議課 | S-T 分析、提問光譜 | 🔥🔥 | Python 算指標 + 畫圖表 |
| F 行政文書 | 企劃、報告、申訴 | 🔥🔥 | 排版好的 `.docx` + 可重複換材料 |
| 顧問人格 / Meta | 教學教練、產提示詞 | 🔥 | 低優先，可淘汰/合併 |

## 五、兩個關鍵洞察（評估時必用）
- **10 變 1**：流程相同、只差材料/類型的多個 Gem（如各種解謎 bot），應**合併成一個**可重複 Skill 傳參數。
- **對照已有 skill 補差**：若已有對應 skill（字幕→audio-to-srt、審題→出題類 skill、教案→備課類 skill、解謎生圖→生圖 skill），遷移＝對照補差，不必重做。

## 六、升級判斷流程（給每個 Gem）
1. 歸到上面 6 類其中之一。
2. 看升級價值（🔥 越多越優先）。
3. 找出「固定（流程＋規格）」與「浮動（材料）」各是什麼。
4. 檢查能否與其他 Gem「10 變 1」合併。
5. 對照是否已有現成 skill。
6. 列出需要的相依工具（查 `deps_matrix.md`）。
7. 產出升級建議：要建什麼 Skill、輸入什麼材料、輸出什麼成品。
